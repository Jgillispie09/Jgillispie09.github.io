To-Do List Application

The .exe file for this application can be downloaded here. The python file can be downloaded here.

Program Overview:

This program is a task management application with a graphical user interface (GUI) designed to enhance user experience. It demonstrates proficiency in Python programming concepts and leverages various modules to provide a functional and user-friendly task management system.

Modules Used:

* tkinter: Used to create a graphical user interface
* simpledialog: Provides input dialog for user interaction
* messagebox: Displays information and confirmation messages
* pickle: Enables saving of the task list, completed list, and task numbering variable
* os: Assists in loading the state at program startup

Python Concepts Utilized:

* Data structures: Lists
* Control flow: If statements, while loops, for loops
* Function usage: Modular functions for each feature
* Indexing: Managing list positions and task numbering
* Variables: Local and global variables for data persistence

How It Works:

* GUI: The application window, buttons, message boxes, and task list display are created utilizing tkinter.
* Button Functionality: Clicking a button triggers its corresponding function to preform a specific task
* For example: Clicking 'Create a New Task' activates the create_task() function

Core Functions:

* create_task(): Prompts the user to enter a new task via simpledialog. If a new task is provided, it is added to the bottom of the list.
* remove_task(): Allows user to remove a task by entering its corresponding number. Users can choose to mark tasks as completed or simply delete them.
* view_list(): Displays tasks present in the task list by using update_showlist(), a function that refreshes the list box display items on the task list.
* view_completed_list(): Displays completed tasks using view_completed_list(), a function that refreshes the list box display to only show completed items.
* delete_list(): Asks user for confirmation via messagebox, if member confirms, task list and completed list are deleted.
* save_state(): Uses pickle to save the task list, completed list, and task numbering. This is called at the end of every major function (create_task() or remove_task()) to ensure automatic saving of the program.
* load_listnum(): Attempts to load listnum.txt for the list numbering variable. If no file exists then listnum will default to 1.

Key Variables:

* task_list: Stores active tasks
* completed: Stores completed tasks
* listnum: Keeps track of task numbering. This number increases or decreases depending on if an item is added or removed from the list.

Task Removal (remove_task()):

This function is a bit more complex than others and I would like to explain how this function works.
* The user inputs the task number they wish to remove.
* The program asks if it should be marked as complete.
* If marked complete, the task is moved to the completed list excluding its number.
* The numbering for the remaining tasks is adjusted to maintain sequential order.
* The updated task list is displayed in the GUI.
* The new state is saved using pickle.
** This function uses indexing to ensure completed items do not carry over their number from the main task list as well as adjusting the list items on the task list to maintain sequential order.
